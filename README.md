# 🕒 Task Time Tracker API

**Task Time Tracker** — сервис учета рабочего времени сотрудников
---

##  Основные возможности

###  Управление задачами 
* **Создание задач** с автоматическим присвоением статуса `NEW` + Переход `NEW` ➡️ `IN_PROGRESS` ➡️ `DONE`.
* **Валидация:** Проверка входных данных.

### Учет временных отрезков 
* **Привязка активности** сотрудника к конкретной задаче.
* **Валидация:** Время начала не может быть позже времени окончания. Нельзя добавить время к несуществующей задаче.

### Расчет времени
* **Расчет минут** работы сотрудника за любой период. Если рабочая смена началась раньше или закончилась позже указанного периода, то отсечет лишнее и посчитает только те минуты, которые попали в искомый интервал.

###  Безопасность и Документация
* **Stateless-авторизация:** Использование **JWT (Bearer Token)**.
* **Защита эндпойнтов:** Все методы бизнес-логики закрыты авторизацией.
* **Swagger UI:** Полная документация для тестирования без сторонних утилит.

## Технологический стек

* **Ядро:** Java 17, Spring Boot 3.5.14
* **Доступ к данным:** MyBatis (Аннотационный подход)
* **Базы данных:** PostgreSQL 15 (Docker), H2 (In-memory)
* **Миграции:** Автоматическая инициализация через `schema.sql`
* **Тестирование:** JUnit 5, Mockito, **Testcontainers** (интеграционные тестов)
* **Документация:** SpringDoc OpenAPI v2

## Конфигурации запуска

Приложение поддерживает два профиля работы, переключаемых через параметр `spring.profiles.active`.

### 🔹 Профиль `dev` (H2 Database)
*Для быстрой проверки кода без настройки окружения.*
* **БД:** В оперативной памяти (сбрасывается при перезапуске).
* **Запуск:** 
  ```bash
  ./mvnw spring-boot:run -Dspring-boot.run.profiles=dev
    ```
* **Консоль H2:** [http://localhost:8080/h2-console](http://localhost:8080/h2-console)

### Профиль `docker` (PostgreSQL)
*Для промышленной эксплуатации и тестирования в реальной среде.*
1.  **Поднимите БД:** 
    ```bash
    docker-compose up -d
    ```
2.  **Запустите приложение:** 
    ```bash
    ./mvnw spring-boot:run -Dspring-boot.run.profiles=docker
    ```
3. **Закрыть докер**
    ```
    docker-compose down -v
    ```

## Обработка ошибок и валидация

`GlobalExceptionHandler`, который возвращает стандартизированный ответ: `{"message": "Текст ошибки"}`.

| HTTP Код | Ошибка | Когда возникает | Пример причины |
| :--- | :--- | :--- | :--- |
| **400** | Bad Request | Ошибка валидации или логики | Поле `title` пустое; `startTime` позже `endTime` |
| **401** | Unauthorized | Отсутствие или неверный токен | Запрос без заголовка `Authorization` |
| **404** | Not Found | Ресурс не найден | Запрос задачи по ID, которого нет в базе |
| **500** | Internal Error | Непредвиденная ситуация | База данных недоступна |

## Как работает 

### Шаг 1: Авторизация
Получите токен для любого пользователя (например, `admin`):
`GET http://localhost:8080/api/auth/token?username=admin`


### Шаг 2: Создание задачи
`POST /api/task`
```json
{
  "title": "Интеграция MyBatis",
  "description": "Настроить мапперы для учета времени"
}
```

### Шаг 3: Фиксация времени
**POST** `/api/time-records`

**Запрос (JSON):**
```json
{
  "employeeId": 55,
  "taskId": 1,
  "startTime": "2026-05-02T10:00:00",
  "endTime": "2026-05-02T12:00:00",
  "description": "Написание SQL запросов"
}
```

### Шаг 4: Получение отчета
Узнаем, сколько минут потратил сотрудник **55** за весь день 2 мая:

**Запрос:**
`GET /api/time-records/employee/55/total?startTime=2026-05-02T00:00:00&endTime=2026-05-02T23:59:59`

**Ответ:** `120` (2 часа в минутах).

### Запуск тестов
Проект покрыт тестами на трех уровнях:

* **Unit-тесты:** Проверка логики расчета минут в `TimeRecordService`.
* **Mock-тесты:** Тестирование контроллеров в изоляции.
* **Integration-тесты:** Запуск полноценного окружения с PostgreSQL в Docker-контейнере через `Testcontainers`.

**Команда для запуска:**
```bash
./mvnw test
```
